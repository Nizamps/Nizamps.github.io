# Nizamuddin PS — Revenue Operations Portfolio

A portfolio of the reporting and operations tooling I built as a founding member of the Sales Operations pilot at **Even Health**. Everything here is **live and clickable** — real reports running in the browser with anonymized sample data.

**Live site:** https://nizamps.github.io/  *(update this link once GitHub Pages is enabled — see below)*

## What's inside

| Project | What it does |
|---|---|
| [**Portfolio site**](index.html) | Single-page overview of my RevOps work, impact, and the systems I shipped. |
| [**RM & CAM Daily Report**](projects/rm-cam-daily-report.html) | Field-ops dashboard across 8 cities and 24 team leads — revenue vs target, HV completion, attendance, with filters and sortable columns. |
| [**Inside Sales Monthly Report**](projects/inside-sales-april-report.html) | Agent productivity and weekly revenue tracking by zone and POD leader, with per-rep revenue sparklines. |
| [**Daily Dials & Talk-Time Report**](projects/daily-dials-report.html) | Hourly calling breakdown with a paste-raw-data parser that auto-detects time slots and ranks top agents. |

## Headline impact

- Home Visit completion lifted **35% → 67%** in four months
- Monthly revenue supported from **Rs 1.5 Cr → Rs 3 Cr**
- Founding Sales Operations function built from **zero to one**
- **239** field reps tracked live across 6+ cities

## Built with

Plain HTML, CSS, and JavaScript — no build step. Each report is a single self-contained file. Underlying work at Even used Google Sheets, Apps Script, Looker Studio, Ameyo, and a custom field-tracking web app.

## A note on the data

All names, employee IDs, emails, and revenue figures in these tools are **anonymized and scrambled**. They preserve the realistic shape and distribution of the original reports without exposing any real person or confidential figure.

## Run locally

Just open `index.html` in any browser. No server or dependencies required.
